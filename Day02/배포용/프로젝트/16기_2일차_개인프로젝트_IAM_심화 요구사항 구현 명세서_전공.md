# [I AM] 나만의 소개 공간을 다양한 페이지로 전환하기 — 전공트랙 (명세서)

> 오늘의 메인: 1일차에서 만든 SPA(단일 페이지) 결과물을 MPA(여러 페이지) 구조로 전환하기
> 사용 도구: Gemini, VS Code, Git, VS Code Extension(Live Server)

---

## 1. 오늘의 목표

1일차에 만든 SPA 결과물을 상단 네비게이션 바로 서로 연결되는 MPA(Multi Page Application) 구조로 재구성합니다.

- 1일차 결과물을 로컬로 가져와 Git으로 버전 관리를 시작
- 페이지를 하나씩 생성
- 확정된 네비게이션 바 코드를 모든 페이지에 동일하게 삽입
- 각 페이지를 조립하고 로컬에서 링크 이동 테스트

**왜 이 방식인가**
- 실제 홈페이지 구조(홈 / 소개 / 갤러리 / 연락처를 별도 페이지로)와 더 비슷합니다.
- 로컬 VS Code에서 작업하며 Git으로 버전 관리하면 기능별 작업 기록을 남길 수 있습니다.

---

## 2. 심화 요구사항 (필수)

### 2-1. Git으로 코드 버전 관리하기
- 1일차 결과물을 로컬로 가져와 Git 저장소로 초기화합니다.
- 기능(페이지/스타일 전환 등) 단위로 커밋을 나누어 기록합니다.

### 2-2. MPA 구성
- 네비게이션 바 컴포넌트를 코드로 먼저 확정합니다.
- 확정된 코드를 모든 페이지에 동일하게 삽입합니다.
- 페이지 수는 2~3개로 제한하는 것을 권장합니다.
- 각 페이지에서 "현재 페이지" 표시가 올바르게 적용되어야 합니다.

### 2-3. Tailwind CSS 적용
- Tailwind CSS를 CDN 방식으로 프로젝트에 추가합니다.
- 기존에 직접 작성한 CSS 클래스를 Tailwind 유틸리티 클래스로 전환합니다.
- 기존 시각적 디자인(색상, 레이아웃, 반응형 동작)은 그대로 유지합니다.

---

## 3. 도전 아이디어 (선택)

- 다크모드 / 라이트모드 토글 버튼
- 스크롤 진행률 상단 바
- 커스텀 마우스 커서
- 배경의 은은한 파티클 애니메이션
- 인터랙티브 타임라인 (학습/커리어 여정)
- 다국어 토글 (한국어/영어)
- 간단한 방명록 (localStorage 사용)
- 게이미피케이션 기능 (미니게임, 이스터에그 찾기 등)
